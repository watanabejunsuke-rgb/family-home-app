# わが家ホーム — 開発引き継ぎドキュメント

家庭向けLINEミニアプリ(家族の日常を管理する「家庭のホーム画面」)。
**エキテンのLINEミニアプリ化PoCの前段検証**という位置づけの個人開発プロジェクト。

## 依頼主について(重要)

- 非エンジニア。GASは自走できる。AI支援前提で開発している
- 報告・説明はすべて日本語で。専門用語は噛み砕く
- 育児中(妻・子2人)。このアプリの最初の実ユーザーは家族
- 金融機能(口座連携・決済・家計簿API等)は**仕様として対象外**。追加しないこと

## 実行方法(ビルド不要)

- この端末には **Node.jsが無い**。npm/Vite/React/Tailwindは使えない。導入提案もしない
- `index.html` をブラウザで開くだけで動く(ES Modules不使用なので file:// でも動作)
- ローカルサーバーで確認する場合: `py -m http.server 8642`(python可・nodeなし)
- データは localStorage(キー: `wagaya-home-v1`)。リセットはメニュー画面の「データを初期化」か `localStorage.clear()`

### ⚠️ キャッシュ対策(ハマりポイント)

ブラウザがJS/CSSを強くキャッシュする。**js/cssを変更したら必ず `index.html` の全参照のバージョンクエリ(`?v=0.2.0`)を上げること**(全19箇所、一括置換でよい)。これを忘れると変更が反映されず延々ハマる(実績あり)。

## アーキテクチャ

Vanilla JS SPA。フレームワークなし。`window.App` 名前空間にすべてぶら下がる。

```
index.html          エントリ。scriptの読み込み順に意味がある(下記の順を守る)
css/tokens.css      デザイントークン(色・余白・角丸・影・文字・z-index・モーション)
css/base.css        リセット+骨格(max-width 480px中央寄せ・セーフエリア・100dvh)
css/components.css  共通部品(card/sheet/toast/nav/fab/chip/task-item/form…)
css/screens.css     画面固有スタイル
js/config.js        APP_CONFIG。LIFF_IDはここ(空=モックモード)
js/icons.js         App.icon(name, size) — アウトラインSVG。絵文字をUIアイコンにしない
js/holidays.js      App.holidayName(dateStr) — 日本の祝日判定(振替休日・国民の休日込み、算出式)
js/store.js         App.store(load/save/update/reset)+モックシード+App.data参照ヘルパー
js/ui.js            App.el(DOM生成)/sheet/toast/confirm/field/chipSelect/taskItem/
                    sectionHeader/emptyState/fab/memberBadges/openStatusSheet
js/liff.js          App.initLiff — LIFF_ID未設定ならモック、設定済ならSDK動的ロード
js/screens/*.js     各画面。App.screens.<route> = { title, nav|back, render(container) }
js/app.js           ハッシュルーティング(#home等)・ヘッダー・下部ナビ・App.go/App.refresh
```

### 画面の追加・変更のルール

1. 画面は `App.screens.<route>` に登録するだけでルーティングされる
   - `nav: "home"` → 下部ナビのタブとして点灯 / `back: true` → 戻るボタン付きサブ画面
2. データ変更は必ず `App.store.update((st) => {...})` 経由。保存と再描画(スクロール位置維持)が自動で走る
3. AI相談画面のみ例外的に `App.store.save()`+手動DOM追加(全再描画するとチャットが跳ねるため)
4. 古いlocalStorageデータに新フィールドが無いケースを必ずガードする(例: `p.careTasks || []`)。シード変更は既存ユーザーには反映されない
5. 削除系は必ず `App.confirm()`(シート型確認)を挟む。操作後は必ず `App.toast()`

## デザイン原則(厳守)

コンセプト: Apple Homeのカード × 無印の余白 × 北欧の温かい色 × Things 3の操作感。
「高機能に見える」より「毎日使っても疲れない」。

- **色は必ずtokens.cssのCSS Variablesから**。ハードコード禁止
  - Background `#F8F7F3` / Primary `#5E7B71` / Accent `#D6B27A` / 文字 `#2B2B2B`(真っ黒禁止)
  - カテゴリ色(family/calendar/task/shopping/plant/note/ai)は**アイコン・ラベル・薄背景のみ**。カード全体を塗らない。1画面で強いアクセントを競合させない
- 余白8px基準 / 角丸16px(大カード20px) / 影は `--shadow-card` 1種のみ(極薄)
- 中央モーダル禁止 → **Bottom Sheet**(`App.sheet`)を使う
- 空状態は全リストに文言付きで用意(「今日は予定なし。ゆっくり過ごせそうです」調)
- エラー文言は技術用語を出さない平易な日本語
- タップ領域44px以上 / aria-label / フォーカスリング維持 / `prefers-reduced-motion` 対応
- LINEグリーン多用禁止・強グラデ禁止・ダッシュボード的な数字詰め込み禁止
