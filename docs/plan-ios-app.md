# iOSアプリ化の検討メモ

「ホーム画面に追加」だけではLINEログインが安定しない問題(下記参照)をきっかけに、iOSネイティブアプリ化を検討している。2026-07-20時点ではまだ着手判断前の情報整理段階。

## 背景: なぜ検討し始めたか

- v0.36.0で「ホーム画面に追加」機能を実装したが、実機検証(2026-07-19〜20)で**iOS Web Clip(ホーム画面に追加したアイコン)からのLINEログインが構造的に不安定**なことが判明した
  - Web Clipは独立した保存領域を持つため、既存のLINEログインセッションを引き継げず、開くたびにログインが必要になりがち
  - ログインのためLINEアプリへ引き継いだ後、**戻ってくる先が元のアイコンではなく新しいSafariタブになってしまう**(2026-07-20、実機で複数回確認。Chrome・Safari両方で再現)。アイコン側は何度タップしても同じログイン画面に迷い込む
  - 対応として、ホーム画面アイコンでは未ログイン時にログイン誘導自体を止め、手元のキャッシュデータのみで表示する形にした(`js/liff.js`のスタンドアロン判定、v0.38.1〜0.38.2)。「編集しても共有されない」ことに気づけるよう案内バナーも追加した(v0.38.2)
  - この対応は「安定して開けること」を優先し「常に最新・常に編集可能であること」は諦めた妥協策。根本解決ではない
- **根本的に解決する方法はネイティブアプリ化**: ネイティブアプリならLINEの**ネイティブiOS SDK**でログインでき、OSレベルで「どのアプリに戻ってくるか」が正しく解決されるため、上記の迷子問題自体が起きなくなる

## 検討の動機(技術的理由以外)

- 上記のログイン問題の根本解決
- 渡辺さんのキャリア形成(iOSアプリのリリース経験)
- キャリアと関係なく、単純にやってみたい・楽しそう、という動機もあり

## 実現可否・必要なもの(2026-07-20時点の整理)

| 項目 | 内容 |
|---|---|
| Apple Developer Program | 年間$99。個人アカウントで可(法人登録は不要)。本人確認に1〜2日 |
| **Mac** | **最大の障壁**。Xcodeは基本的にMac専用で、現在の開発環境(Windows)では不可。**新規に用意する必要がある**(2026-07-20時点で未確保) |
| サーバー | **新規には不要**。既存のGAS+スプレッドシートのバックエンドをそのまま流用できる。LINEログインだけネイティブSDKに差し替え、pull/push等の既存APIはそのまま呼べる想定 |
| LINE側の追加設定 | 既存のLINEログインチャネルに「iOSプラットフォーム」を追加登録するだけ。新規チャネル作成は不要 |

### Mac確保の選択肢

| 選択肢 | 目安コスト | 所感 |
|---|---|---|
| Mac mini(新品、M4・base構成) | 約10万円前後 | 一番現実的。プロジェクト専用出費にならず汎用的に使える |
| 中古Mac(Intel/M1世代) | 約3〜6万円 | 予算重視ならアリ。ビルドはやや遅い |
| クラウドMac(MacinCloud等) | 月$20〜30 | 初期費用は抑えられるが月額が積み重なる。対話的な開発にはやや不向き |

### 実装方針の選択肢

1. **フルネイティブ書き直し(Swift/SwiftUI)**: 現状のJS資産(7000行超)を全部作り直すことになり非現実的
2. **WKWebView + ネイティブSDKでログインだけ差し替え(有力案)**: 既存のWeb版コードをほぼそのまま流用し、Swift側は「WebViewを表示する」「LINEログインだけネイティブでやってJSに結果を渡す」という薄い橋渡し(数十〜百数十行程度)で済む。**Node.js不要**で、このプロジェクトの「ビルド不要・Node無し」方針とも相性が良い
3. Capacitor等のハイブリッドツール: Node.js環境が要るため今回は非採用の方向

### 配布方法の選択肢

利用者が実質2人(渡辺さん・パートナー)であることを踏まえ、以下の三段階で考える。**どの段階から始めても、Apple Developer登録・Xcodeプロジェクト作成・LINEネイティブSDKでのログイン橋渡しという中核作業は共通**なので、後から配布方法だけ格上げできる(作業の手戻りはほぼない)。

1. **Xcodeから自分の端末へ直接インストール**: 審査なし、一番早い。Macがあれば即日試せる
2. **TestFlight**: 軽い審査はあるが数日で通ることが多い。メール招待で最大1万人まで配布可能
3. **App Store公開**: ストア掲載用の画像・プライバシーポリシー・審査対応が別途必要。**「単なるWebサイトのラッパー」とみなされ審査に落ちるリスク**(Appleガイドライン4.2 最低限の機能性)が実際にある。キャリア形成・経験目的なら挑戦する価値はある
